# № 1
## Публичный интерфейс множества:
- bstInsert (добавить новый эл. в мн-во, еще можно назвать add())
- bstContains (проверить есть ли эл. в мн-ве, еще можно назвать has())
- bstSize (мощность мн-ва)
- bstMin (мин. эл. в мн-ве)
- bstMax (макс. эл. в мн-ве)
- bstDelete (удалить эл. из мн-ва, еще можно назвать remove())
- bstMerge (обьединение мн-в)
- bstFree (освобождение памяти)
## Детали реализации:
- bstInorder
- bstPreorder
- bstPostorder
- bstHeight
- Весь итератор
- bstIsValid
- bstKthMin

# № 2
## Функции не имеющие смысла в реализации множества:
- bstInorder, bstPreorder, bstPostorder (функции для обхода дерева на множестве бесполезны)
- bstHeight (у множества нет характеристики "высота")
- bstKthMin (у множества нет понятия k-й элемент)

# № 3
## Что нужно изменить, чтобы можно было заменить BST на хеш-таблицу:
- В BST есть поле root, в хэш-таблице его нет, нужно сделать доступ к данным только через функции
- удалить функции bstInorder, bstPreorder, bstPostorder, bstHeigh, bstKthMin, bstIsValid, bstMin, bstMax
- Имена функций и аргументы в новой реализации должны совпадать со старой
